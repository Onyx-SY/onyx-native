# Debug 技能

当用户报告一个 bug 时，按以下系统化流程调试：

## 1. 复现（Reproduce）
- 确认 Bug 的具体表现：什么输入、什么环境、什么输出
- 尝试用相同条件复现
- 如果无法复现，用 `choose_ask` 让用户补充复现步骤
- 记录复现成功率（必现/偶发）

## 2. 定位（Root Cause）
- 查看相关错误日志/堆栈
- 用 `grep_search` 搜索相关错误模式
- 检查最近的代码变更（`git log`/`git diff`）
- 提出假设 → 验证 → 排除，直到找到根因
- 根因确认后，用一句话总结给用户

## 3. 修复（Fix）
- 评估修复方案的风险
- 高风险修改先用 `submit_plan` 提交计划
- 低风险直接修复
- 每处修改后简要说明改了什么、为什么

## 4. 验证（Verify）
- 确认修复后 Bug 不再复现
- 检查是否引入了新的问题（回归测试）
- 告知用户修复结果

## 原则
- 一次只改一个问题，不要顺手修无关的
- 如果 15 分钟内没定位到根因，告知用户当前进度和下一步计划
- 某些 Bug 可能是配置/环境问题，先排除这些再改代码
