# Refactor 技能

当需要重构代码时（优化结构、提升可读性、减少重复），按以下流程操作：

## 1. 分析现状（Analyze）
- 理解当前代码的功能和外部接口（不改变行为）
- 识别问题点：重复代码、过长函数、职责混杂、命名模糊
- 用 `choose_ask` 确认用户对重构范围的期望

## 2. 制定计划（Plan）
- 列出要改动的文件和改动内容
- 标注每个改动的风险等级
- 改动涉及 3+ 个文件时，用 `submit_plan` 提交计划

## 3. 逐步重构（Execute）
- **一次只做一个重构操作**，不要混合多个目的
- 每个操作后运行测试/语法检查
- 常见的重构操作：
  - 提取函数（Extract Method）
  - 重命名（Rename）
  - 移动文件（Move）
  - 合并重复代码（Merge Duplicates）
- 每步完成后用 `mark_step_complete` 标记

## 4. 验证行为不变（Verify）
- 重构的铁律：**不改行为**
- 运行所有可用测试
- 检查语法和类型
- 对比重构前后的输出是否一致

## 原则
- 不要在一处重构时顺手修 Bug（分开提交）
- 小步提交，每步都可回退
- 如果某步导致测试失败，立即回退并分析原因
- 用户关心的不是你怎么重构的，而是重构后代码更好用了
