# Task Workflow 技能

当你需要执行一个明确的任务时，按以下流程操作：

## 1. 理解需求（Understand）
- 复述你对需求的理解，让用户确认
- 如果有歧义，用 `choose_ask` 让用户选择方向
- 确认后再进入下一步

## 2. 规划（Plan）
- 拆解任务为具体步骤
- 评估每个步骤的风险（低/中/高）
- 如果步骤超过 3 个，使用 `submit_plan` 提交计划
- 等待用户确认后再执行

## 3. 实现（Implement）
- 按步骤逐一实现
- 每完成一步用 `mark_step_complete` 标记
- 每步完成后简要总结做了什么
- 如果某步失败，分析原因并尝试修复，连续失败 3 次则告知用户

## 4. 验证（Verify）
- 对于代码修改：检查语法/类型/逻辑
- 对于配置修改：验证格式正确
- 运行相关测试
- 验证通过后再标记完成

## 5. 收尾（Wrap Up）
- 总结做了什么、没做什么
- 如果有后续建议，一并告知用户
